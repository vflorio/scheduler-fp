# Supervisor

## Dependency Graph

The service does not depend directly on external tools. Instead, it relies on generic interfaces that can be easily swapped out (e.g. for testing).

```mermaid
graph TD
    Service["Supervisor Service"]

    Service --> AB["AndroidBridge"]
    Service --> MD["mDNS Discovery"]
    Service --> DR["Device Registry"]
    Service --> NT["Notification"]

    AB -. impl .-> adb["adb"]
    MD -. impl .-> avahi["avahi"]
    DR -. impl .-> suitest["suitest"]
    NT -. impl .-> slack["slack"]

    DR -- populates --> DB["Internal Device DB"]

    style Service fill:#4a90d9,color:#fff
    style AB fill:#f5a623,color:#fff
    style MD fill:#f5a623,color:#fff
    style DR fill:#f5a623,color:#fff
    style NT fill:#f5a623,color:#fff
    style adb fill:#7b8a8b,color:#fff
    style avahi fill:#7b8a8b,color:#fff
    style suitest fill:#7b8a8b,color:#fff
    style slack fill:#7b8a8b,color:#fff
    style DB fill:#2ecc71,color:#fff
```

| Interface | Implementation | Purpose |
|---|---|---|
| **AndroidBridge** | `adb` | Communicate with Android devices |
| **mDNS Discovery** | `avahi` | Discover devices on the local network |
| **Device Registry** | `suitest` | Fetch registered lab devices, populate the internal Device DB |
| **Notification** | `slack` | Send alerts and status notifications |
